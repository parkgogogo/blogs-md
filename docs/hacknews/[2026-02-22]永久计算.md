## 超越开源的开放计算，由设计为可重组的部件组成。

可延展系统设计用于支持任意重组，可跨环境重复使用，并可自由与他人共享。修改此类系统发生在使用情境中，而非通过某些单独的工具和技能。

与单体应用相比，这些无定形的计算机由操作者在其实践过程中精心调整和打造。

可延展系统的核心是由能够组合的异质部件组成。在计算情境中，它是不同可互操作系统共享的属性，类似于通过[Unix管道](unix.html#pipes)通信的程序、文件浏览器和一些基于流的编程环境。

可延展部件是变换器链中的一个变换器，其本身也由同样相互连接的链接、节点或代理组成。系统越可延展，这些部件的粒度区分就越模糊。

	可延展性

 孤岛
 节点强制捕获消息的系统即为孤岛。

 浅层
 消息可以进入和离开一个本身不可延展的系统，这是一种表面的可延展性。

 平均
 消息可以在可延展系统的内部节点之间自由流动，但能力受限于领域、界面或操作者能力。

 深层
 消息可以无缝传递到系统的每个层级，层级划分已完全消失。

这种粒度的一个潜在优势是复杂性的分布，使操作者能够更温和地过渡到了解构成问题解决方案的更精细部分。

为可逆性而设计是使系统能够被各种能力水平的人接近，降低错误成本，并通过支持可撤销的转换来限制对操作者的指责。

一些自动机在其转换中保存了足够的状态，使计算可以反向进行，而无需用大量内存来存储截图。

可逆计算的常见例子是数字乘以[分数](fractions.html)并使用倒数恢复该数字，或堆栈机操作rot。位运算符not或堆栈机swap是[对合的](reversible_computing.html)，这意味着再次应用相同的运算符可以恢复先前的状态。

为连接而设计是使用一种无点的消息传递范式，不识别其操作的参数。此处提及它是因为它可能在深度可延展系统中被证明是有价值的，作为结合可检查性到机械转换层面和合理评估性能的一种方式。

: SQUARED ( n -- n^2 ) DUP * ;
: CUBED ( n -- n^3 ) DUP SQUARED * ;

在这样的系统中，节点仅组合其他节点，其中包括组合子，它们操作参数而不考虑上下文。这是一种花哨的说法，即连接性子表达式可以用代表该相同子表达式的名称替换。

: POPCNT ( n -- bits ) 0 swap
BEGIN dup WHILE tuck 1 AND + swap 1 rshift REPEAT
DROP ;

一个最大可访问的系统，使得位移动的更精细细节是可观察的，但不在通用硬件上运行，这将因其不可访问性而显得自相矛盾。因此，根据宿主计算机的不同，像上面的函数定义（可能作为单个操作码存在）应该仍然是可观察系统定义的一部分，即使它不被执行。

(add ?x ?y) -> (?x + ?y)
(?x ?y add) -> (?x + ?y)

((?a) + ?b) -> (?a + (?b))
(0 + ?b) -> (sum ?b)

(add 1 2)

上面程序的前两条规则都支持前缀和后缀到中级表示法的翻译，以及转录英语符号add，否则可以进行本地化。这是为了演示 vastly different 系统之间的互操作是可能的。

## 弄清楚如何最好地利用已经存在的数百万台设备。

抢救计算认为，计算机产品生命周期的终点应被视为庆祝的时刻，是其社会经济背景终于可以被回收的时刻。可拾荒电子零件是那些不再生产，但在垃圾填埋场中以数十亿计的可用零件。那些能够利用低技术工具从拾荒零件中创建新设计的人将能够保存电子学。

它不主张回到过去，尽管主张大幅减少人造能源的使用，但相信人类的创造力能够将问题转化为解决方案，将竞争转化为合作，将[废物转化为资源](solarpunk.html)。

为封装而设计：为确保例如README文件可以被任何用户阅读，分发磁盘包含一个简单的文本编辑程序，可以显示README文件。尽管大多数用户已经在他们的系统上有一个，软件供应商不应假设情况会如此。

	当文件使用PKZIP压缩时，需要解压程序（如PKUNZIP）来展开文件。然而，PKZIP中的一个选项允许将简单版本的解压程序与每个压缩文件捆绑在一起。选择此选项会创建一个可执行文件，运行时自动展开为原始文件，避免了压缩文件的接收者是否手边有适当解压软件的问题。

## 利用今天计算能力的充裕，为未来基础设施崩溃做准备。

崩溃计算优先考虑社区需求，旨在为知识共享做出贡献，以通过[基础设施崩溃](collapse.html)维持计算实践，它是一种以将被丢弃的东西转化为更新资源的眼光来接触被丢弃事物的实践。

### 失败场景

- 供应：由于新产品不再生产或无法获得，难以获得新设备和外设。

- 电力：电力供应间歇性，或被路由以满足比为计算机供电更关键的需求。

- 连接性：由于基础设施退化或地缘政治原因禁止，互联网访问是间歇性的。

- 过时：由于文档缺失或不兼容的外设和软件导致无法运行。

- 计划性淘汰：由于过期证书、复制保护或其他人为手段导致无法运行。

	电影[《银色星球》(1988)](https://www.imdb.com/title/tt0093593/)中的场景

为下降而设计确保系统对间歇性能源供应和网络连接具有弹性。无需生产新产品，也无需处理电子垃圾。如果你的新软件不再在旧硬件上运行，它比旧软件[更差](obsolescence.html)。软件应在现有硬件上运行，并依赖模块化以启用多样化的组合和实现。它是关于重新发明基本工具，使它们可访问、可扩展、坚固、模块化、易于维修且有良好文档。
