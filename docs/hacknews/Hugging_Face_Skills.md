# Hugging Face Skills

# Hugging Face 技能

Hugging Face Skills are definitions for AI/ML tasks like dataset creation, model training, and evaluation. They are interoperable with all major coding agent tools like OpenAI Codex, Anthropic's Claude Code, Google DeepMind's Gemini CLI, and Cursor.

Hugging Face 技能是用于 AI/ML 任务的定义，例如数据集创建、模型训练和评估。它们可与所有主流编码代理工具互操作，如 OpenAI Codex、Anthropic 的 Claude Code、Google DeepMind 的 Gemini CLI 和 Cursor。

The Skills in this repository follow the standardized format [Agent Skill](https://agentskills.io/home) format.

此仓库中的技能遵循标准化的 [Agent Skill](https://agentskills.io/home) 格式。

## How do Skills work?

## 技能是如何工作的？

In practice, skills are self-contained folders that package instructions, scripts, and resources together for an AI agent to use on a specific use case. Each folder includes a `SKILL.md` file with YAML frontmatter (name and description) followed by the guidance your coding agent follows while the skill is active.

在实践中，技能是自包含的文件夹，将指令、脚本和资源打包在一起，供 AI 代理在特定用例中使用。每个文件夹包含一个 `SKILL.md` 文件，其中包含 YAML 前置元数据（名称和描述），然后是您的编码代理在技能激活时遵循的指导。

> [!NOTE]
> 'Skills' is actually an Anthropic term used within Claude AI and Claude Code and not adopted by other agent tools, but we love it! OpenAI Codex uses an `AGENTS.md` file to define the instructions for your coding agent. Google Gemini uses 'extensions' to define the instructions for your coding agent in a `gemini-extension.json` file. **This repo is compatible with all of them, and more!**

> [!NOTE]
> "技能（Skills）"实际上是 Anthropic 在 Claude AI 和 Claude Code 中使用的术语，并未被其他代理工具采用，但我们很喜欢它！OpenAI Codex 使用 `AGENTS.md` 文件来定义编码代理的指令。Google Gemini 使用"扩展（extensions）"在 `gemini-extension.json` 文件中定义编码代理的指令。**此仓库与所有这些工具兼容，甚至更多！**

> [!TIP]
> If your agent doesn't support skills, you can use [`agents/AGENTS.md`](agents/AGENTS.md) directly as a fallback.

> [!TIP]
> 如果您的代理不支持技能，您可以直接使用 [`agents/AGENTS.md`](agents/AGENTS.md) 作为后备方案。

## Installation

## 安装

Hugging Face skills are compatible with Claude Code, Codex, Gemini CLI, and Cursor.

Hugging Face 技能与 Claude Code、Codex、Gemini CLI 和 Cursor 兼容。

### Claude Code

### Claude Code

1. Register the repository as a plugin marketplace:  
   
   将此仓库注册为插件市场：

```
/plugin marketplace add huggingface/skills
```

2. To install a skill, run:  
   
   要安装技能，请运行：
   
```
/plugin install <skill-name>@huggingface/skills
```

For example:  

例如：

```
/plugin install hugging-face-cli@huggingface/skills
```

### Codex

### Codex

1. Codex will identify the skills via the `AGENTS.md` file. You can verify the instructions are loaded with:

   Codex 将通过 `AGENTS.md` 文件识别技能。您可以通过以下命令验证指令是否已加载：

```
codex --ask-for-approval never "Summarize the current instructions."
```

2. For more details, see the [Codex AGENTS guide](https://developers.openai.com/codex/guides/agents-md).

   更多详情，请参阅 [Codex AGENTS 指南](https://developers.openai.com/codex/guides/agents-md)。

### Gemini CLI

### Gemini CLI

1. This repo includes `gemini-extension.json` to integrate with the Gemini CLI.

   此仓库包含 `gemini-extension.json` 以与 Gemini CLI 集成。

2. Install locally:  

   本地安装：

```
gemini extensions install . --consent
```

or use the GitHub URL:

或使用 GitHub URL：

```
gemini extensions install https://github.com/huggingface/skills.git --consent
```

4. See [Gemini CLI extensions docs](https://geminicli.com/docs/extensions/#installing-an-extension) for more help.

   更多帮助请参阅 [Gemini CLI 扩展文档](https://geminicli.com/docs/extensions/#installing-an-extension)。

通过 Cursor 插件流程从仓库 URL（或本地检出）安装。

   在您的编码代理中重新安装或重新加载技能包，以便更新的文件夹可用。

## 批判性思考评论

Hugging Face 的这个 Skills 仓库代表了一个重要的行业趋势：AI/ML 工具正在从孤立的命令行工具向更加集成化的"技能"生态系统演进。

**积极方面：**

1. **标准化努力值得赞赏** — 采用 Agent Skill 格式并兼容多个主流编码代理（Claude Code、Codex、Gemini CLI、Cursor），这是减少碎片化的一大步。Hugging Face 作为 AI 开源社区的核心玩家，推动这种标准化具有很大的影响力。

2. **降低使用门槛** — 通过 `SKILL.md` 封装复杂的 ML 工作流程，让不熟悉 Hugging Face 生态的开发者也能快速上手。特别是对于模型训练、评估等专业性较强的任务，这种封装大大降低了认知负担。

3. **生态整合** — 从提供的技能列表可以看出，Hugging Face 正在构建一个端到端的 ML 工作流：从数据集创建、模型训练、评估到论文发布，形成闭环。这种一站式体验对研究人员和开发者都很有吸引力。

**值得思考的方面：**

1. **"技能"概念的混淆** — 文档自己也承认，"Skills"是 Anthropic 的术语，其他厂商使用不同的术语（Codex 用 AGENTS.md，Gemini 用 extensions）。虽然 Hugging Face 声称兼容所有格式，但这种术语不一致可能会导致用户困惑。行业何时能达成真正的标准化？

2. **供应商锁定风险** — 虽然 Skills 仓库支持多个代理工具，但它明显是以 Hugging Face Hub 为中心的。这可能会强化 Hugging Face 作为模型托管和协作平台的垄断地位。对于希望保持多云或自托管策略的团队，这种便利性可能带来依赖风险。

3. **技能的维护和版本控制** — 文档提到可以通过 `./scripts/publish.sh` 重新生成清单，但没有详细说明技能的版本控制策略。当底层 API 发生变化时，如何确保旧版本技能仍然可用？如何管理技能之间的依赖关系？

4. **安全考量** — 技能可以执行 Hugging Face CLI 操作、管理仓库、运行云计算任务。如果技能的定义被恶意篡改，可能会导致严重的安全问题。文档中没有提及安全审核机制或权限最小化原则。

**总体评价：**

Hugging Face Skills 是一个实用且及时的创新，它顺应了 AI 辅助编程的大趋势，将 Hugging Face 的 ML 能力无缝集成到开发者的日常编码环境中。对于已经在 Hugging Face 生态中的用户来说，这无疑会提升效率。但对于更广泛的 AI 社区而言，技能/扩展/代理指令的标准化仍然是一个需要持续关注的开放问题。
